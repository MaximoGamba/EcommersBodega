================================================================================
  DOCUMENTACIÓN DEL PROYECTO (API REST + JWT + JPA)
  Carpeta: docs/
================================================================================

Este conjunto de archivos .txt resume la estructura y el trabajo realizado en el
proyecto Spring Boot, ordenado por capas.

Archivos incluidos:

  01-entidades.txt
      Modelo de dominio JPA: tablas, relaciones, enums y notas sobre seguridad
      en la serialización JSON.

  02-repositorios.txt
      Interfaces Spring Data JPA, consultas derivadas y WineSpecification.

  03-servicios.txt
      Lógica de negocio: interfaces, implementaciones y DTOs internos (payload).

  04-controladores.txt
      Endpoints REST, DTOs de entrada/salida y reglas de acceso resumidas.

  05-seguridad-y-excepciones.txt
      JWT, SecurityConfig, manejo global de errores y respuestas 401/403.

  06-entrega-webservices-persistencia-stock-auth.txt
      Texto de entrega: Web Services, persistencia, stock en catálogo,
      registro y login (JWT).

  07-jwt-uso-e-implementacion.txt
      JWT: configuración, emisión, cabecera Bearer, filtro, 401/403 y archivos
      involucrados.

  08-prueba-local-auth-categoria.txt
      Evidencia de prueba local: register/authenticate/login + intento de
      create category con token JWT.

  09-diagnostico-jwt-categories.txt
      Diagnóstico técnico del 401 en create category y hallazgo de esquema
      en tabla categories.

  10-diagnostico-create-vino-data-integrity.txt
      Causa raíz de DATA_INTEGRITY en POST /vinos (FK de clasificación,
      especialmente azucar_id).

  11-diagnostico-token-create-vino.txt
      Prueba local de JWT en create vino: el token valida, falla la
      serialización de entidades LAZY en la respuesta.

  12-diagnostico-user-puede-crear-categoria.txt
      Causa de autorización: un USER autenticado puede crear categorías por
      falta de restricción de rol en el endpoint POST /categories.

Paquete base del código: com.uade.tpo.demo
